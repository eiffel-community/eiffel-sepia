Test Orchestration

The Test Orchesration actor role is responsible for triggering and/or canceling test case executions, and grouping them into test suites. This may be done directly based on some internal logic or static test case lists, or it may be informed by Test Case Execution Recipes created by the Test Execution Recipe Creation role.

None as of yet.
