Flow Context Definition

The Flow Context Definition role creates and publishes EiffelFlowContextDefinedEvent, which other actors may then refer to in the events they publish in order to tag them with various types of context. This in turn enables sorting and filtering of Eiffel events. A simple implementation of the Flow Context Definition role merely consists of an API and/or GUI where the relevant properties of the context object can be entered, whereupon the correct event syntax is formed and published. There is little need for internal storage or management of flow context data, as the events themselves can be used as a data source.
